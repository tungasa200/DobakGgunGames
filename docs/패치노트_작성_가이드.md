# 패치노트 작성 가이드

> 어드민 페이지(`/admin/patch-notes`)에서 패치노트를 작성할 때 참고하는 형식 가이드.
> 유저가 보는 페이지(`/patch-notes`)에 바로 게시되므로 작성 후 내용을 꼭 검토할 것.

---

## 1. 기본 작성 원칙

1. **유저 관점으로 작성** — 코드 변수명, 파일명, 내부 용어는 쓰지 않는다. "AppleCanvas 컴포넌트 수정" ❌ → "사과게임 드래그 오류 수정" ✅
2. **무엇이 바뀌었는지만** — 원인 분석, 구현 방식 같은 기술적 세부사항은 `버그수정이력.md`에 기록하고 패치노트에는 넣지 않는다.
3. **짧고 명확하게** — 항목 하나당 한 줄. 길어지면 유저가 읽지 않는다.
4. **게임 카테고리를 정확히 선택** — 전체 공지(서비스 점검, 신규 기능 등)가 아니면 반드시 해당 게임을 선택한다.

---

## 2. 버전 규칙

`주버전.부버전.패치` 형식을 사용한다.

| 번호 | 올리는 기준 | 예시 |
|------|------------|------|
| **주버전** | 서비스 구조가 크게 바뀌는 경우 (신규 게임 추가, 인증 시스템 도입 등) | `1.0.0` → `2.0.0` |
| **부버전** | 게임 플레이에 영향을 주는 신기능 추가, 기존 기능 변경 | `1.1.0` → `1.2.0` |
| **패치** | 버그 수정, 스타일 수정, 소소한 개선 | `1.2.0` → `1.2.1` |

> 현재 버전 기준: 서비스 공개 초기이므로 `0.x.x`로 시작한다.

---

## 3. 게임 카테고리 선택 기준

| 카테고리 | 언제 선택하나 |
|----------|-------------|
| **전체 공지** | 서비스 전체에 영향 (서버 점검, 신규 게임 출시, 회원 시스템 변경 등) |
| **공통** | 특정 게임이 아닌 공통 기능 (랭킹 시스템, 로그인, UI 공통 개선 등) |
| **💣 지뢰찾기** | 지뢰찾기 전용 수정/업데이트 |
| **⚾ 숫자야구** | 숫자야구 전용 수정/업데이트 |
| **🟦 블록폴** | 블록폴 전용 수정/업데이트 |
| **🃏 솔리테어** | 솔리테어 전용 수정/업데이트 |
| **🍎 사과게임** | 사과게임 전용 수정/업데이트 |

> 여러 게임이 동시에 영향을 받으면 **공통**으로 올리거나, 게임별로 따로 올린다.

---

## 4. 제목 작성 규칙

- 가장 핵심적인 변경 사항 하나를 뽑아 한 줄로 요약한다.
- 버그 수정만 있으면 "~수정", 기능 추가면 "~ 추가/개선" 으로 끝낸다.

```
✅ 모바일 터치 오류 수정
✅ 랭킹 등록 개선 및 UI 버그 수정
✅ 게임오버 깃발 표시 개선
✅ 서버 기반 점수 검증 시스템 도입

❌ fix: 사과게임 모바일 세로 모드 시작 즉시 크래시 수정  (커밋 메시지 그대로 쓰지 말 것)
❌ AppleCanvas 드래그 패시브 이벤트 오류 수정  (파일명 노출)
❌ 여러 게임 버그를 수정했습니다  (너무 모호)
```

---

## 5. 내용 작성 형식

아래 섹션을 필요한 것만 조합해서 사용한다. 없는 섹션은 생략한다.

```
[버그수정]
- 항목 1
- 항목 2

[개선]
- 항목 1

[업데이트]
- 항목 1
```

### 섹션 구분

| 섹션 | 사용 기준 |
|------|---------|
| `[버그수정]` | 잘못 동작하던 것을 고쳤을 때 |
| `[개선]` | 동작은 했지만 더 나아졌을 때 (성능, UX 등) |
| `[업데이트]` | 기존 기능을 변경하거나 새 기능을 추가했을 때 |

---

## 6. 작성 예시

### 예시 A — 버그 수정 위주 (게임별)

```
제목: 모바일 오류 수정 및 플리커 개선
버전: 0.3.1
카테고리: 🍎 사과게임

[버그수정]
- 모바일 세로 모드에서 게임 시작 시 크래시 수정
- 드래그 중 화면이 순간 깜빡이는 현상 수정
- 세로 모드에서 순위 등록이 안 되는 오류 수정

[개선]
- 모바일 터치 감도 개선
```

---

### 예시 B — 기능 업데이트 (공통)

```
제목: 랭킹 등록 안정화
버전: 0.4.0
카테고리: 공통

[버그수정]
- 세션 생성 실패 시 랭킹 등록이 불가능한 문제 수정 (전 게임 적용)

[업데이트]
- 랭킹 등록 모달에 로그인된 닉네임 자동 입력
```

---

### 예시 C — 전체 공지

```
제목: 서버 기반 점수 검증 시스템 도입
버전: 0.5.0
카테고리: 전체 공지

[업데이트]
- 점수 조작을 방지하기 위해 서버 검증 시스템이 적용되었습니다
- 적용 게임: 지뢰찾기, 솔리테어, 사과게임
- 이전 방식으로 등록된 기록은 유지됩니다
```

---

## 7. 자주 하는 실수

| 실수 | 올바른 방법 |
|------|-----------|
| 커밋 메시지를 그대로 붙여넣기 | 유저 관점의 언어로 다시 작성 |
| 한 패치노트에 모든 내용 몰아넣기 | 게임별로 나누거나 버전을 분리 |
| 개발 용어 사용 (`컴포넌트`, `API`, `null`, `리팩터링`) | 유저가 이해할 수 있는 말로 바꾸기 |
| 버그 원인·수정 방법 상세 설명 | `버그수정이력.md`에 기록, 패치노트엔 결과만 |
| 카테고리 `전체 공지`로 통일 | 해당 게임 카테고리를 정확히 선택 |

---

## 8. 과거 커밋 → 패치노트 변환 예시

아래는 실제 커밋 내역을 패치노트 항목으로 바꾼 예시다.

| 커밋 메시지 | 패치노트 항목 |
|------------|-------------|
| `fix: 게임오버 시 잘못 놓인 깃발 ❌로 표시` | 게임오버 시 잘못 놓은 깃발이 ❌로 표시되도록 개선 |
| `fix: 웨일 사이드바에서 우클릭 깃발이 첫 번째만 동작하는 버그 수정` | 웨일 브라우저에서 깃발 우클릭이 첫 번째만 적용되는 오류 수정 |
| `fix: 블록폴 게임오버 후 모달 반복 출현 버그 수정` | 게임오버 화면이 반복해서 뜨는 오류 수정 |
| `fix: 사과 제거 시 순간 깜빡임(플리커) 수정` | 사과를 지울 때 화면이 순간 깜빡이는 현상 수정 |
| `update: 사과 색 좀더 선명하게 변경` | 사과 색상을 더 선명하게 조정 |
| `fix: 세션 생성 실패 시 랭킹 등록 불가 문제 개선 (전 게임 적용)` | 간헐적으로 랭킹 등록이 안 되는 문제 수정 (전 게임 적용) |
| `fix: 지뢰찾기 모바일 꾹터치 시 물음표 먼저 나오는 버그 수정` | 모바일에서 깃발 꾹터치 시 물음표가 먼저 나오는 오류 수정 |
