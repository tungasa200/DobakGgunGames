## 2단계: 배포 환경 구성

### 2-1. Railway (백엔드 + MySQL)

1. [railway.app](https://railway.app) 회원가입 → Hobby Plan ($5/월)
2. **New Project → Deploy from GitHub** → `tungasa200/DobakGgunGames`
3. Root Directory: `backend`
4. 환경변수 설정 (Railway 대시보드):
   ```
   DB_HOST     = (Railway MySQL 내부 호스트)
   DB_PORT     = 3306
   DB_NAME     = dobakggun
   DB_USER     = (Railway MySQL 유저)
   DB_PASS     = (Railway MySQL 비밀번호)
   HMAC_SECRET = (랜덤 문자열 32자 이상)
   CORS_ORIGINS = https://your-domain.com
   ```
5. **Add Plugin → MySQL** → DB 자동 프로비저닝
6. 커스텀 도메인: Settings → Networking → Custom Domain → `api.도메인.com`

### 2-2. Vercel (프론트엔드)

1. [vercel.com](https://vercel.com) → New Project → GitHub Import
2. Root Directory: `frontend`
3. Framework Preset: Vite
4. 환경변수 설정:
   ```
   VITE_API_URL     = https://api.도메인.com
   VITE_HMAC_SECRET = (Railway에 설정한 것과 동일)
   ```
5. 커스텀 도메인: Project Settings → Domains → `www.도메인.com`

### 2-3. DNS 설정 (커스텀 도메인 보유 시)

| 레코드 | 이름 | 값 |
|---|---|---|
| CNAME | www | `cname.vercel-dns.com` |
| CNAME | api | Railway 제공 도메인 |

## 4단계: 엑셀 UI 모드 구현 ✅ 완료

### 아키텍처

원본 `excel-games/common/excel-ui.js` 방식(HTML 주입)을 React Context + 컴포넌트로 재구현.

```
GamePage (excel=true)
  └─ ExcelShell (타이틀바·리본·수식바·시트탭·상태바 렌더)
       └─ TetrisBoard / MinesweeperBoard / ... (excel={true})
            └─ useExcelShell() 훅으로 수식바·상태바 업데이트
```

- **ExcelShellContext** — 게임 컴포넌트 → 셸로 상태 푸시 (수식바 셀·수식, 상태바 항목)
- **ExcelShell** — 전체 레이아웃 + 홈 드롭다운 + 시트탭 관리
- **게임 컴포넌트** — `useExcelShell()`로 수식바/상태바 업데이트, 리본 게임 그룹은 prop으로 ExcelShell에 전달

### 4-1. 구현 파일

| 파일 | 역할 |
|---|---|
| `components/excel/ExcelShellContext.tsx` | Context + `useExcelShell()` 훅 |
| `components/excel/ExcelShell.tsx` | 전체 레이아웃 (타이틀바·리본·수식바·시트탭·상태바) |
| `components/excel/ExcelShell.module.css` | Excel 크롬 스타일 |

### 4-2. 게임별 Excel 연동

각 게임 보드(`excel={true}` 시) `useExcelShell()`을 호출해 셸 UI 업데이트:

| 게임 | 수식바 | 상태바 |
|---|---|---|
| 테트리스 | `=TETRIS_DROP(level,{lv},speed,{ms}ms)` | 점수·레벨·줄 |
| 지뢰찾기 | `=MINESWEEPER_FLAG(mines,{n})` | 지뢰·타이머 |
| 숫자야구 | `=BASEBALL_TRY(attempt,{n})` | 시도 횟수 |
| 사과게임 | `=APPLE_SCORE(sum,{n})` | 점수·남은 시간 |
| 솔리테어 | `=SOLITAIRE_MOVE(count,{n})` | 이동 수·시간 |

## 4-4 스타일 세부 수정 (게임별 미완성 항목) 🔄 진행 중

### 작업 수칙

1. **게임 한 개씩 순차 진행** — 한 게임이 완전히 완료된 뒤 다음 게임으로 넘어갈 것
2. **원본과 1:1 대조** — `excel-games/[game]/index.html`, `excel.html` 을 직접 열어 CSS/HTML 구조를 확인하고 맞출 것. 패턴으로 일괄 수정하지 말 것
3. **엑셀 모드 셀 크기 주의** — 게임마다 셀 크기·스타일이 다름. 배경 그리드(`cellSize` prop)와 게임 셀 크기를 반드시 일치시킬 것
4. **엑셀 랭킹·룰 탭** — 거의 미구현 상태. 원본 `excel.html`의 `excel-ranking-sheet`, `excel-rules-sheet` 구조를 참고하여 ExcelShell 시트 탭과 연동할 것
5. **원본과 달라도 괜찮은 것** — 일반 모드 하단의 엑셀 버튼은 NormalHeader에 통합되어 있으므로 생략 가능

### 리본 게임 그룹 아키텍처 변경 (이번 작업에서 도입)

기존에 `ExcelShell`의 prop `ribbonGameGroup`으로 전달하던 방식을 **Context 기반**으로 변경함.

- `ExcelShellContext`에 `ribbonGameGroup: ReactNode` + `setRibbonGameGroup` 추가
- 게임 컴포넌트 내부에서 `useExcelShell().setRibbonGameGroup(...)` 으로 push
- `ExcelShell` prop에서 `ribbonGameGroup` 제거

이후 게임들도 동일 패턴으로 리본 그룹을 등록하면 됨.
