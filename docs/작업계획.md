# DobakGgun Games 작업계획

## 목표

`excel-games/` (PHP + 순수 JS 원본)를 `dobakggun/` (React + Spring Boot)으로 재구축.

## 2단계: 배포 환경 구성

### 2-1. Railway (백엔드 + MySQL)

1. [railway.app](https://railway.app) 회원가입 → Hobby Plan ($5/월)
2. **New Project → Deploy from GitHub** → `tungasa200/DobakGgunGames`
3. Root Directory: `backend`
4. 환경변수 설정 (Railway 대시보드):
   ```
   DB_HOST     = (Railway MySQL 내부 호스트)
   DB_PORT     = 3306
   DB_NAME     = dobakggun
   DB_USER     = (Railway MySQL 유저)
   DB_PASS     = (Railway MySQL 비밀번호)
   HMAC_SECRET = (랜덤 문자열 32자 이상)
   CORS_ORIGINS = https://your-domain.com
   ```
5. **Add Plugin → MySQL** → DB 자동 프로비저닝
6. 커스텀 도메인: Settings → Networking → Custom Domain → `api.도메인.com`

### 2-2. Vercel (프론트엔드)

1. [vercel.com](https://vercel.com) → New Project → GitHub Import
2. Root Directory: `frontend`
3. Framework Preset: Vite
4. 환경변수 설정:
   ```
   VITE_API_URL     = https://api.도메인.com
   VITE_HMAC_SECRET = (Railway에 설정한 것과 동일)
   ```
5. 커스텀 도메인: Project Settings → Domains → `www.도메인.com`

### 2-3. DNS 설정 (커스텀 도메인 보유 시)

| 레코드 | 이름 | 값 |
|---|---|---|
| CNAME | www | `cname.vercel-dns.com` |
| CNAME | api | Railway 제공 도메인 |

## 3단계: 게임 마이그레이션

각 게임을 React 컴포넌트 + 커스텀 훅으로 이식.
원본 로직은 `excel-games/[game]/common.js` 참고.

### 마이그레이션 순서 (복잡도 낮은 순)

#### 3-1. 숫자야구 ⚾
- 원본: [excel-games/numberBaseball/common.js](../../excel-games/numberBaseball/common.js)
- 목표 파일:
  - `frontend/src/games/baseball/useBaseballGame.ts` — 순수 로직 훅
  - `frontend/src/games/baseball/BaseballBoard.tsx` — UI 컴포넌트
- 특이사항: 애니메이션 없음, 가장 간단

#### 3-2. 지뢰찾기 💣
- 원본: [excel-games/minesweeper/common.js](../../excel-games/minesweeper/common.js)
- 목표 파일:
  - `frontend/src/games/minesweeper/useMinesweeperGame.ts`
  - `frontend/src/games/minesweeper/MinesweeperBoard.tsx`
- 특이사항: 셀 상태 그리드 → `useReducer` 활용

#### 3-3. 사과게임 🍎
- 원본: [excel-games/apple/common.js](../../excel-games/apple/common.js)
- 목표 파일:
  - `frontend/src/games/apple/useAppleGame.ts`
  - `frontend/src/games/apple/AppleCanvas.tsx`
- 특이사항: `<canvas>` + `requestAnimationFrame` — `useRef` + `useEffect` 활용

#### 3-4. 솔리테어 🃏
- 원본: [excel-games/solitaire/common.js](../../excel-games/solitaire/common.js)
- 목표 파일:
  - `frontend/src/games/solitaire/useSolitaireGame.ts`
  - `frontend/src/games/solitaire/CardBoard.tsx`
- 특이사항: 드래그앤드롭 카드 상태 머신, 두 번째로 복잡

#### 3-5. 테트리스 🟦
- 원본: [excel-games/tetris/common.js](../../excel-games/tetris/common.js)
- 목표 파일:
  - `frontend/src/games/tetris/useTetrisGame.ts`
  - `frontend/src/games/tetris/TetrisCanvas.tsx`
- 특이사항: T-스핀 감지, lock delay, hold, canvas 루프 — 가장 복잡, 마지막

### 게임 페이지 라우팅 연결

마이그레이션 완료된 게임부터 [GamePage.tsx](../frontend/src/pages/GamePage.tsx)에 연결:

```tsx
// frontend/src/pages/GamePage.tsx
const GAME_COMPONENTS = {
  baseball:    { normal: BaseballPage, excel: BaseballExcelPage },
  minesweeper: { normal: MinesweeperPage, excel: MinesweeperExcelPage },
  // ...
}
```

## 4단계: 엑셀 UI 모드 구현

### 4-1. ExcelShell 컴포넌트

원본 `excel-games/common/excel-ui.js`의 HTML 주입 방식을 React로:

```tsx
// frontend/src/components/excel/ExcelShell.tsx
<ExcelShell gameName="지뢰찾기">
  <ExcelRibbon groups={ribbonGroups} />
  <ExcelFormulaBar cellName="B7" formula="=COUNTA(A3:A7)" />
  <ExcelSheetArea>
    {sheet === 'game'    && <MinesweeperBoard />}
    {sheet === 'ranking' && <RankingSheet />}
  </ExcelSheetArea>
  <ExcelSheetTabs sheets={['게임','랭킹','룰']} active={sheet} onChange={setSheet} />
  <ExcelStatusBar />
</ExcelShell>
```
