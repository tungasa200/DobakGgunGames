# 패치노트 아카이브 작업계획

## 개요

사용자에게 서비스 업데이트 내역을 보여주는 패치노트 아카이브 페이지를 구현한다.  
패치노트 데이터는 DB에 저장하며, 목록 페이지(5개 단위 페이지네이션) + 상세 페이지 2개로 구성한다.  
관리자 작성 UI는 별도 작업으로 보류 — 현재는 DB 직접 INSERT로 운영.

---

## 사용자 플로우

```
/patch-notes 접속
  └─ 기본 탭: 전체 (최신 5개 제목 목록 표시)
       ├─ 탭 전환: 공통 / 지뢰찾기 / 숫자야구 / 블록폴 / 솔리테어 / 사과게임
       ├─ 제목 클릭 → /patch-notes/:id (상세 페이지)
       └─ [더보기] 버튼 클릭 → 다음 5개 추가 로드 (hasNext인 경우에만 노출)
```

## 데이터 구조

### PatchNote 엔티티

| 필드 | 타입 | 설명 |
|------|------|------|
| `id` | Long (PK) | 자동 증가 |
| `version` | VARCHAR(20) | 버전 문자열 (예: `v1.2.3`) |
| `title` | VARCHAR(200) | 패치노트 제목 |
| `content` | TEXT | 본문 내용 (마크다운) |
| `game` | ENUM | 카테고리 (아래 참고) |
| `createdAt` | DATETIME | 작성일시 |

### PatchNoteGame Enum

| 값 | 의미 |
|----|------|
| `ALL` | 전체 탭에서만 노출 (모든 게임에 해당하는 내용) |
| `COMMON` | 공통 (게임 무관 서비스 전반 업데이트) |
| `MINESWEEPER` | 지뢰찾기 |
| `BASEBALL` | 숫자야구 |
| `BLOCKFALL` | 블록폴 |
| `SOLITAIRE` | 솔리테어 |
| `APPLE` | 사과게임 |

> **전체 탭 동작**: game 필터 없이 전체 조회. 나머지 탭은 해당 game 값으로 필터링.

## API 설계

### 목록 조회

```
GET /api/patch-notes?game={game}&page={page}&size=5

Query Parameters:
  game  : ALL | COMMON | MINESWEEPER | ... (생략 시 전체 조회)
  page  : 0-based 페이지 번호 (기본값 0)
  size  : 페이지당 항목 수 (고정 5)

Response:
{
  "content": [
    {
      "id": 12,
      "version": "v1.2.3",
      "title": "블록폴 난이도 밸런스 조정",
      "game": "BLOCKFALL",
      "createdAt": "2026-04-20T12:00:00"
    },
    ...
  ],
  "hasNext": true,
  "totalCount": 23
}
```

### 상세 조회

```
GET /api/patch-notes/{id}

Response:
{
  "id": 12,
  "version": "v1.2.3",
  "title": "블록폴 난이도 밸런스 조정",
  "content": "## 변경사항\n- ...",
  "game": "BLOCKFALL",
  "createdAt": "2026-04-20T12:00:00"
}
```


### API 래퍼 (`src/api/patchnotes.ts`)

```ts
export type PatchNoteGame =
  'ALL' | 'COMMON' | 'MINESWEEPER' | 'BASEBALL' | 'BLOCKFALL' | 'SOLITAIRE' | 'APPLE';

export type PatchNoteItem = {
  id: number;
  version: string;
  title: string;
  game: PatchNoteGame;
  createdAt: string;
};

export type PatchNoteDetail = PatchNoteItem & {
  content: string;
};

export type PatchNoteListResponse = {
  content: PatchNoteItem[];
  hasNext: boolean;
  totalCount: number;
};

export const patchNotesApi = {
  getList: (game?: PatchNoteGame, page = 0): Promise<PatchNoteListResponse> => { ... },
  getDetail: (id: number): Promise<PatchNoteDetail> => { ... },
};
```

### 목록 페이지 UI 구성 (`PatchNotesPage.tsx`)

```
[전체] [공통] [지뢰찾기] [숫자야구] [블록폴] [솔리테어] [사과게임]  ← 탭

┌──────────────────────────────────────────────┐
│ BLOCKFALL  v1.2.3                2026-04-20  │
│ 블록폴 난이도 밸런스 조정              →    │  ← 클릭 시 /patch-notes/12
└──────────────────────────────────────────────┘
┌──────────────────────────────────────────────┐
│ MINESWEEPER  v1.2.2              2026-04-15  │
│ 지뢰찾기 모바일 버그 수정              →    │
└──────────────────────────────────────────────┘
... (최대 5개)

                  [더보기]   ← hasNext가 true일 때만 노출
```

**상태 관리:**

| 상태 | 초기값 | 설명 |
|------|--------|------|
| `activeTab` | `'ALL'` (전체) | 선택된 탭 |
| `items` | `[]` | 로드된 패치노트 목록 |
| `page` | `0` | 현재 페이지 번호 |
| `hasNext` | `false` | 더보기 버튼 노출 여부 |
| `loading` | `false` | 로딩 상태 |

**탭 전환 시:** `items` 초기화 + `page` 0으로 리셋 + 재조회
