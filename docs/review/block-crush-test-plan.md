## 2. 엣지 케이스 테스트 (PRD §8 기반, 18개)

> PRD §8 확정 전 작성된 선행 시나리오. PRD §8 수신 시 케이스 ID와 전제 조건을 동기화할 것.

각 케이스 형식: **전제 조건 → 조작 → 기대 결과**

### EC-01: 보드 완전 비어있을 때 1×1 블록 배치

- **전제 조건**: 8×8 보드 완전 비어 있음. 트레이에 1×1 블록 존재
- **조작**: (0,0) 셀에 1×1 블록 드래그앤드롭
- **기대 결과**: 배치 성공. 점수 +1. 클리어 없음. 보드 (0,0) 셀만 채워짐
- **우선순위**: P2 High

### EC-02: 보드 거의 가득 찬 상태에서 마지막 1칸에 1×1 배치 후 8칸 완성

- **전제 조건**: 특정 행에 7칸 채워짐. 트레이에 1×1 블록 존재
- **조작**: 해당 행의 남은 1칸에 1×1 블록 배치
- **기대 결과**: 배치 성공. 해당 행 클리어 발동. 점수 = 배치 1점 + 100점(1줄) + 콤보 보너스(해당 콤보 카운트 × 50)
- **우선순위**: P1 Critical

### EC-03: 가로·세로 동시 완성 교차점에 블록 배치

- **전제 조건**: 특정 행에 7칸, 특정 열에 7칸 채워짐. 교차하는 셀 1개 비어 있음. 트레이에 1×1 블록 존재
- **조작**: 교차 셀에 1×1 블록 배치
- **기대 결과**: 행·열 동시 클리어(2줄). 점수 = 배치 1점 + 200점(2줄 기본) + 다중 보너스 100점 + 콤보 보너스. 교차점의 셀은 두 번 제거되지 않고 1번만 처리
- **우선순위**: P1 Critical

### EC-04: 3블록 중 2개 배치 완료 후 남은 1개가 유일하게 배치 가능한 위치

- **전제 조건**: 트레이에 1개 블록만 남은 상태. 보드에 해당 블록이 들어갈 공간이 정확히 1곳 존재
- **조작**: 드래그 중 해당 공간 이외 위치에 드롭 시도 → 이후 올바른 위치에 드롭
- **기대 결과**: 잘못된 위치 시도 → 거부 + 원위치. 올바른 위치 → 배치 성공. 3슬롯 모두 비면 새 3블록 지급
- **우선순위**: P2 High

### EC-05: 점수 0점 상태에서 게임 오버

- **전제 조건**: 게임 시작 직후. 어떤 블록도 배치하지 않음
- **조작**: 게임 오버 조건을 인위적으로 유발 (개발 도구 사용 또는 상황 재현)
- **기대 결과**: 점수 0으로 게임 오버 모달 표시. 점수 0 등록 가능(API가 score=0 허용하는지 확인). 점수 0점이 랭킹에 등록되어도 API 400 반환 없음
- **우선순위**: P2 High

### EC-06: 콤보 카운트 최대 연속 달성 후 클리어 없는 배치

- **전제 조건**: comboCount = 5 상태 (5회 연속 클리어 성공)
- **조작**: 클리어가 발생하지 않는 위치에 블록 배치
- **기대 결과**: 해당 배치에서 콤보 보너스 없음(이미 comboCount=5는 직전 클리어 시 반영됨). 배치 직후 comboCount 0으로 리셋. 다음 배치에서 콤보 보너스 미가산
- **우선순위**: P2 High

### EC-07: 동일 행을 두 번 완성하는 블록 배치 불가 시도

- **전제 조건**: 특정 행이 이미 클리어된 직후 보드 상태
- **조작**: 해당 행(현재 비어있음)의 일부 셀에 새 블록 배치 시도
- **기대 결과**: 클리어된 행은 빈 상태이므로 새 블록 배치 가능. 특이 동작 없음 (클리어 즉시 해당 행의 상태가 정상적으로 비어있어야 함)
- **우선순위**: P2 High

### EC-08: 타이머 또는 턴 제한이 없는 것 확인 (무한 대기)

- **전제 조건**: 게임 진행 중
- **조작**: 블록을 드래그하지 않고 5분 이상 대기
- **기대 결과**: 시간 제한 없음. 게임 자동 종료 없음. 5분 후에도 블록 배치 가능 상태 유지 (PRD에 타이머 없다는 전제)
- **우선순위**: P3 Medium

### EC-09: 보드 전체가 채워진 상태

- **전제 조건**: 8×8 = 64셀 전체 채워진 상태
- **조작**: 다음 블록 지급 시도
- **기대 결과**: 어떤 블록도 배치 불가 → 즉시 게임 오버. 단, 새 블록 지급 전에 게임 오버를 판정하거나 새 블록 지급 후 배치 불가 확인 후 게임 오버 — PRD §4.4 플로우 순서 확인 필요
- **우선순위**: P1 Critical

### EC-10: 5×1 블록(I_5_H)이 8칸 행에서 중간에 배치되어 나머지 3칸 분리

- **전제 조건**: 특정 행에 I_5_H 블록이 배치됨. 행의 0~4번 열 채워짐
- **조작**: 해당 행의 5~7번 열 3칸을 다른 블록으로 채우려 시도
- **기대 결과**: 3칸짜리 블록(예: I_3_H)이 있으면 배치 후 전체 8칸 완성 → 클리어. 2칸+1칸 블록 조합도 동일하게 클리어 가능
- **우선순위**: P2 High

### EC-11: 3×3 블록(SQ_3)이 보드 모서리에 배치

- **전제 조건**: 보드의 (0,0)~(2,2) 영역이 비어 있음
- **조작**: SQ_3 블록을 (0,0) 기준점으로 배치
- **기대 결과**: 배치 성공(9점). 해당 영역 채워짐. 행/열 클리어 없음 (열 0,1,2 각 3칸밖에 안 찼음)
- **우선순위**: P2 High

### EC-12: 블록을 드래그한 채 화면 밖(브라우저 경계)으로 이동 후 드롭

- **전제 조건**: 게임 화면에서 블록 드래그 중
- **조작**: 마우스를 브라우저 창 밖으로 이동 후 마우스 버튼 해제
- **기대 결과**: 블록 트레이 원위치 복귀. 드래그 상태 초기화. 게임 상태 손상 없음
- **우선순위**: P2 High

### EC-13: 새 트레이 지급 직후 배치 가능한 블록이 하나도 없는 경우

- **전제 조건**: 보드가 복잡하게 채워져 있고 새로 지급된 3개 블록이 모두 현재 보드에 배치 불가
- **조작**: 새 트레이 지급 이후 상태 관찰
- **기대 결과**: 배치 가능 여부 판정이 즉시 실행되어 게임 오버 발동. 지급 직후 프레임에서 판정 수행 필요. 배치 불가 블록이 보드 위에 "떠다니는" 현상 없음
- **우선순위**: P1 Critical

### EC-14: 동일 콤보 연산에서 배치 점수 + 줄 클리어 점수 + 콤보 보너스 모두 동시 가산

- **전제 조건**: comboCount = 2 상태
- **조작**: 블록 배치 후 1줄 클리어 발생
- **기대 결과**: 배치 점수 + 100점(1줄) + 100점(2×50 콤보) 모두 단일 배치 이벤트에서 합산 가산. 분리 시차 없음
- **우선순위**: P1 Critical

### EC-15: 같은 배치로 가로 3줄 동시 클리어 (L자 또는 정사각형 블록 특수 케이스)

- **전제 조건**: 3개 행이 각각 7칸 채워진 상태. 블록 배치 1회로 3행 동시 완성 가능
- **조작**: 해당 블록 배치
- **기대 결과**: 3줄 동시 클리어. PRD §5.3 다중 보너스 테이블 기준 점수 적용. 3줄 이상 동시 클리어 처리 로직 정상 동작
- **우선순위**: P2 High

### EC-16: 랭킹 등록 폼에서 닉네임 미입력 후 제출

- **전제 조건**: 게임 오버. 닉네임 입력 폼 표시 중
- **조작**: 닉네임 필드를 비우고 제출 버튼 클릭
- **기대 결과**: 제출 거부. "닉네임을 입력해 주세요" 등 안내 메시지 표시. API 미호출
- **우선순위**: P2 High

### EC-17: 랭킹 등록 중 네트워크 타임아웃

- **전제 조건**: 게임 오버 후 닉네임 입력. 네트워크 불안정 상태 시뮬레이션 (DevTools Network Throttle: Offline)
- **조작**: 점수 등록 제출
- **기대 결과**: 등록 실패. 사용자에게 오류 안내 메시지 표시. 폼이 닫히지 않고 재시도 가능 상태 유지
- **우선순위**: P2 High

### EC-18: 세션 만료 후 점수 등록 시도

- **전제 조건**: 세션 발급 후 만료 시간(PRD §4에서 확인)이 경과한 상태
- **조작**: 점수 등록 API 호출
- **기대 결과**: 400 또는 401 응답. "세션 만료" 안내. 등록 실패
- **우선순위**: P2 High

## 3. 점수 계산 정밀 검증

> PRD §5 공식 기준 직접 계산. PRD 확정 후 아래 공식 적용 여부 재확인 필수.
>
> **PRD §5 확정 공식:**
> - 배치 점수 = 블록 셀 수 × 1점
> - 줄 클리어 기본 점수 = 클리어된 줄 수 × 100점
> - 다중 보너스 (PRD §5.3 확정 테이블): 1줄=0 / 2줄=+100 / 3줄=+300 / 4줄=+600 / 5줄=+1000 / 6줄=+1500 / 7줄=+2100 / 8줄=+2800
> - 콤보 보너스 = comboCount × 50점 (클리어 발생 전 comboCount 기준 — PRD §5.5 확정)
> - 콤보 리셋: 클리어 없는 배치 후 comboCount = 0

### 시나리오 A: I_4_H 배치 → 1줄 클리어, 콤보 0 (첫 클리어)

- **전제 조건**: comboCount = 0 상태. I_4_H(가로 1×4 블록) 배치 후 해당 행 8칸 완성
- **계산**:
  - 배치 점수: 4셀 × 1 = **4점**
  - 줄 클리어(1줄): 1 × 100 = **100점**
  - 다중 보너스: 1줄이므로 **0점**
  - 콤보 보너스: comboCount=0 → 클리어 발생으로 comboCount=1 증가하나, 보너스 가산은 증가 이전 값(0) 기준 또는 증가 후 값(1) 기준인지 PRD §5 확인 필수
    - PRD §5 "콤보 보너스 = comboCount × 50" 기준으로 클리어 전 comboCount=0이면 콤보 보너스 **0점**
  - **기대 합계: 4 + 100 + 0 + 0 = 104점**
- **검증**: 배치 후 점수 UI가 정확히 +104점 증가하는지 확인

### 시나리오 B: SQ_3(3×3) 배치 → 가로 1줄 + 세로 1줄 동시 클리어, 콤보 1

- **전제 조건**: comboCount = 1 상태 (직전 배치에서 1번 클리어). SQ_3 배치 후 특정 행·열 동시 완성
- **계산**:
  - 배치 점수: 9셀 × 1 = **9점**
  - 줄 클리어(2줄): 2 × 100 = **200점**
  - 다중 보너스: 2줄 이상이므로 **+100점**
  - 콤보 보너스: comboCount=1 × 50 = **50점**
  - **기대 합계: 9 + 200 + 100 + 50 = 359점**
- **검증**: 배치 후 점수 UI가 정확히 +359점 증가하는지 확인

### 시나리오 C: I_5_H 배치 → 클리어 없음, 이전 콤보 3

- **전제 조건**: comboCount = 3 상태 (직전 3회 연속 클리어). I_5_H 배치 후 클리어 없음
- **계산**:
  - 배치 점수: 5셀 × 1 = **5점**
  - 줄 클리어: 0줄 = **0점**
  - 다중 보너스: **0점**
  - 콤보 보너스: 클리어 없으므로 보너스 없음 = **0점**
  - comboCount 리셋: 0으로 초기화
  - **기대 합계: 5 + 0 + 0 + 0 = 5점**
- **검증**: 배치 후 점수 UI가 정확히 +5점 증가. comboCount UI가 0으로 표시

### 시나리오 D: I_4_V(세로 4×1 블록) 배치 → 세로 1줄 클리어, 콤보 2

- **전제 조건**: comboCount = 2 상태. I_4_V 배치 후 해당 열 8칸 완성
- **계산**:
  - 배치 점수: 4셀 × 1 = **4점**
  - 줄 클리어(1줄 — 세로): 1 × 100 = **100점**
  - 다중 보너스: 1줄 = **0점**
  - 콤보 보너스: comboCount=2 × 50 = **100점**
  - **기대 합계: 4 + 100 + 0 + 100 = 204점**
- **검증**: 배치 후 점수 UI가 정확히 +204점 증가하는지 확인

### 시나리오 E: SQ_2(2×2 블록) 배치 → 가로 2줄 + 세로 1줄 동시 클리어(3줄), 콤보 0

- **전제 조건**: comboCount = 0 상태. SQ_2 배치 후 2개 행 + 1개 열 동시 완성(3줄 동시 클리어 가능한 보드 세팅)
- **계산**:
  - 배치 점수: 4셀 × 1 = **4점**
  - 줄 클리어(3줄): 3 × 100 = **300점**
  - 다중 보너스: 3줄 → PRD §5.3 확정값 **+300점**
  - 콤보 보너스: comboCount=0 = **0점**
  - **기대 합계: 4 + 300 + 300 + 0 = 604점**
- **검증**: 배치 후 점수 UI가 정확히 +604점 증가하는지 확인

## 10. 반려 기준 체크리스트

다음 조건 중 하나라도 해당되면 PR 반려 + 담당 developer에게 차단 메시지:

- [ ] PRD에 Excel 모드 지시가 있었는데 일반 모드만 구현된 경우
- [ ] FT-22: 3블록 모두 배치 불가인데 게임 오버가 발동되지 않는 경우
- [ ] FT-15: 배치 점수(셀 수 × 1점)가 잘못 계산되는 경우
- [ ] FT-18: 콤보 보너스(comboCount × 50점)가 잘못 계산되는 경우
- [ ] API-07: 분당 3건 초과 등록 요청이 429 없이 통과되는 경우
- [ ] API-08: 동일 sessionId 재사용이 성공하는 경우 (세션 재사용 공격 취약)
- [ ] SEC-01 / SEC-02: HMAC 위조 또는 score 변조가 서버에서 통과되는 경우
- [ ] SEC-03 / SEC-04: XSS / SQL Injection 입력이 처리 없이 통과되는 경우
- [ ] REG-01 ~ REG-06: 기존 게임 기능 회귀 파손 발견 시
- [ ] P1 Critical 버그 미해결 상태로 완료 요청
