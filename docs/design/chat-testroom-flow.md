## 1. 접근 정책 요약

| 유저 상태 | Test Lab 섹션 표시 | `/dbgchat` 진입 |
|---|---|---|
| 비로그인 (게스트) | **숨김** | 라우트 가드 → `/login` |
| 로그인 + USER 등급 | 표시 (버튼 클릭 시 차단 페이지) | 라우트 가드 → 차단 페이지 (URL 유지) |
| 로그인 + FRIEND 등급 | 표시 | 정상 진입 |
| 로그인 + ADMIN 등급 | 표시 | 정상 진입 |

- 등급명("도박꾼", "FRIEND") 은 사용자에게 노출하지 않는다.
- `FriendRoute` 가드는 비로그인 시 `/login` 리다이렉트, USER 등급 시 URL 유지하며 차단 페이지 렌더.

## 3. 화면 1 — 홈 페이지 Test Lab 섹션

### 3-1. 위치 및 레이아웃

- `HomePage.tsx` 의 기존 `.grid` 내 마지막 카드 (빈 카드 `.card` 요소)
- 기존 게임 카드와 동일한 `.card` 컨테이너 재사용
- 카드 크기: `minmax(280px, 1fr)` — 그리드 정책 그대로

### 3-2. 카드 구성 와이어프레임

```
┌─────────────────────────────────────────┐
│  🧪  Test Lab                           │  <- .cardHeader (기존 구조 그대로)
│      [nameKo 없음 — "Test Lab" 고정]     │
├─────────────────────────────────────────┤
│                                         │
│   현재 실험 중인 기능들을 미리 체험할   │  <- 설명 텍스트 (13px, #64748b)
│   수 있는 공간입니다.                   │
│                                         │
│   ┌─────────────────────────────────┐   │
│   │  [채팅 아이콘] 실시간 채팅 랩   │   │  <- 진입 버튼 (Link to="/dbgchat")
│   └─────────────────────────────────┘   │
│                                         │
└─────────────────────────────────────────┘
```

### 3-3. 진입 버튼 명세

- 라벨: `실시간 채팅 랩`
- 아이콘: `💬` (버튼 라벨 왼쪽)
- 컴포넌트: `<Link to="/dbgchat">` — 일반 앵커 링크 (라우트 가드는 진입 후 처리)
- 스타일: 기존 `.btn .btnNormal` 패턴 준용 (배경 white, 테두리 #2c3e50, 호버 시 반전)
- 너비: 버튼이 카드 전체 너비를 채우도록 `width: 100%`
- 높이: 최소 44px (모바일 터치 타겟)
- 비로그인 시: 섹션 전체를 렌더하지 않음 (`user` 존재 시에만 Test Lab 카드 노출)

### 3-4. 노출 조건 정리

```
user === null  → Test Lab 카드 미렌더 (숨김)
user !== null  → Test Lab 카드 렌더
                  (FRIEND/USER 구분 없이 버튼 표시)
                  (USER가 클릭하면 /dbgchat 에서 FriendRoute가 차단 처리)
```

### 4-2. 방 만들기 UI 결정: 인라인 폼 (토글 방식)

**결정: 별도 모달이 아닌 인라인 폼 (목록 상단 슬라이드 다운)**

근거:
- 방 이름 한 필드만 입력하는 단순한 폼이므로 모달 오버레이 오버엔지니어링
- 모바일에서 모달 키보드 UX 불편 — 인라인 폼이 자연스러운 스크롤 유지
- 목록과 폼의 맥락 전환 없이 동일 화면에서 완결 (방 생성 완료 즉시 방 진입)
- 기존 프로젝트가 모달 패턴을 사용하지 않음 (코드베이스 일관성)

**동작 순서:**
1. `+ 방 만들기` 버튼 클릭 → `CreateRoomForm` 슬라이드 다운 (또는 즉시 표시)
2. 방 이름 입력 (1~30자) + `방 만들기` 버튼 클릭
3. 성공 → 응답받은 `roomId` 로 즉시 `/dbgchat/{roomId}` 이동
4. 실패 → 폼 하단 인라인 에러 메시지, 폼 유지

### 4-3. 인라인 폼 와이어프레임

```
┌──────────────────────────────────────────────────┐
│  방 이름                                          │
│  ┌──────────────────────────────────┐ ┌────────┐ │
│  │ 방 이름을 입력하세요 (최대 30자)  │ │방 만들기│ │
│  └──────────────────────────────────┘ └────────┘ │
│  [에러 메시지 — 빨간색 12px, 아래 표시]            │
│                               0 / 30              │
└──────────────────────────────────────────────────┘
```

- 카드 형태 (`background: white; border: 1px solid #e2e8f0; border-radius: 12px; padding: 16px 20px`)
- 방 만들기 버튼: 액센트 컬러 (`background: #2c3e50; color: white`) — 전송 중 `disabled + 스피너`
- 문자 카운터: 입력창 우하단 `{현재}/{최대}`, 28자 초과 시 주황색, 30자 초과 불가 (서버는 30자로 trim)
- `Enter` 키: 폼 제출
- `Escape` 키: 폼 닫힘
- 취소 링크 (`✕ 취소`) — 버튼 오른쪽 또는 폼 우상단

### 4-4. 방 목록 카드 와이어프레임

```
┌────────────────────────────────────────────────────┐
│  🗨  [방 이름 — 최대 1줄 ellipsis]                  │
│      개설: [creatorNick]  · 최근 활동: [상대 시각]  │
│                                             [입장 →] │
└────────────────────────────────────────────────────┘
```

- 카드 전체가 클릭 가능 (`cursor: pointer`)
- 카드 호버: `border-color: #aa3bff` 전환 (150ms)
- 방 이름: `font-size: 15px; font-weight: 600; color: #0f172a`
- 메타 텍스트 (개설자/활동시각): `font-size: 12px; color: #94a3b8`
- 최근 활동 시각: 상대 시간 표현 (예: "방금 전", "5분 전", "1시간 전")
  - 1시간 이내: 상대 시간
  - 1시간 이상: 절대 시각 `MM/DD HH:mm`
- `입장 →` 텍스트: `font-size: 12px; color: #aa3bff; font-weight: 600`
- 카드 클릭 시 → `/dbgchat/{roomId}` 이동

### 4-5. 빈 상태 (방이 없을 때) 와이어프레임

```
┌──────────────────────────────────────────────────┐
│                                                  │
│              💬                                  │
│                                                  │
│        아직 열린 채팅방이 없습니다.               │
│        첫 번째 방을 만들어 보세요.                │
│                                                  │

│                                                  │
└──────────────────────────────────────────────────┘
```

- 아이콘: `font-size: 40px`
- 타이틀: `font-size: 16px; font-weight: 600; color: #0f172a`
- 설명: `font-size: 13px; color: #94a3b8`
- 방 만들기 버튼: 헤더 영역의 `+ 방 만들기`와 같은 동작 (폼 펼침)

### 4-6. Degraded 상태 (Redis 장애 시 `degraded: true`)

- 목록 상단에 경고 배너: `⚠ 일시적인 서버 오류로 목록이 정확하지 않을 수 있습니다.`
- 배경: `#fffbeb; border: 1px solid #fcd34d; border-radius: 8px; padding: 10px 14px`
- 텍스트: `font-size: 13px; color: #92400e`
- 방 만들기 버튼은 비활성화 (503 응답 예상되므로) → 버튼 위에 툴팁: `서버 점검 중입니다.`

### 4-7. 반응형 레이아웃

| 브레이크포인트 | 레이아웃 |
|---|---|

| 모바일 (< 480px) | 패딩 12px, 방 카드 단일 컬럼, 헤더/폼 세로 스택 |

```
┌──────────────────────────────────────────────────────┐
│  ← 목록   💬 [방 이름 — 40자까지 표시, ellipsis]     │
│                                           ● 연결됨   │

```

- `← 목록`: `<Link to="/dbgchat">` — `font-size: 13px; color: #64748b`

- `ConnectionStatus` 배지: 오른쪽 정렬
- 헤더 하단: `border-bottom: 1px solid #e2e8f0`
- 배경: `#ffffff`
- 패딩: `12px 16px`

### 5-3. 히스토리 로딩 → 실시간 수신 순서

```
1. 페이지 마운트
        ↓
2. REST GET /api/chat/rooms/{roomId}/history
        ↓
   성공: 히스토리 메시지 배열 설정 (오래된 것 위, 최신이 아래)
   실패 (404): "채팅방이 종료되었습니다." 토스트 + /dbgchat 이동
   degraded: true: 빈 배열 + 상단 경고 배너 표시
        ↓
3. STOMP 연결 (ConnectionStatus = "연결 중")
        ↓
4. STOMP CONNECT 성공
        ↓
   ConnectionStatus = "연결됨"
   SUBSCRIBE /topic/room/{roomId}
   SUBSCRIBE /user/queue/errors
        ↓
5. 새 메시지 수신 → messages 배열 끝에 추가 + 자동 스크롤 맨 아래
        ↓
6. 연결 끊김 감지 → ConnectionStatus = "재연결 중"
   3회 자동 재연결 시도 (간격: 2s → 4s → 8s 지수 백오프)
        ↓
   성공: ConnectionStatus = "연결됨" 복원
   최종 실패: "서버와 연결이 끊어졌습니다." 토스트 + /dbgchat 이동
```

### 5-4. 메시지 목록 + 메시지 버블 와이어프레임

**CHAT 타입 — 타인 메시지 (왼쪽 정렬)**
```
         [닉네임 13px #64748b]
  ┌──────────────────────────┐  [시각 11px #94a3b8]
  │  메시지 텍스트           │
  └──────────────────────────┘
  배경: #f1f5f9, 테두리 없음, radius 0 12 12 12
```

**CHAT 타입 — 내 메시지 (오른쪽 정렬)**
```
                     [시각 11px #94a3b8]
     ┌──────────────────────────┐
     │  메시지 텍스트           │
     └──────────────────────────┘
  배경: #aa3bff (accent), 텍스트 white, radius 12 0 12 12
```

**SYSTEM 타입 — 중앙 정렬**
```
        ─────  익명테스터님이 입장하셨습니다.  ─────
  배경 없음, color #94a3b8, font-size 12px, 좌우 padding, 선으로 양쪽 구분
```

- 내 메시지 판별: `message.userId === currentUser.id`
- 닉네임 표시: 타인 메시지에만 표시 (내 메시지는 생략)
- 타임스탬프: 메시지 버블 오른쪽/왼쪽 하단 (locale 시각 — `HH:mm` 형식)
- 날짜 구분선: 날짜가 바뀌면 `─── 2026년 4월 23일 ───` 구분선 삽입
- 동일 발신자의 연속 메시지: 닉네임/아바타 숨기고 버블만 (간격 4px)
- 첫 메시지 또는 발신자 변경: 닉네임 표시 (간격 12px)

### 5-5. 자동 스크롤 정책

- 새 메시지 수신 시:
  - 이미 맨 아래(또는 맨 아래에서 100px 이내)에 있으면 → **자동으로 맨 아래 스크롤**
  - 위로 스크롤 중인 경우 → **자동 스크롤 하지 않음** + `↓ 새 메시지 N개` 배지 표시
- `↓ 새 메시지` 배지 클릭 → 맨 아래 스크롤 + 배지 사라짐
- 히스토리 최초 로딩 완료 후: 즉시 맨 아래 스크롤

### 5-6. ChatInput 와이어프레임

```
┌──────────────────────────────────────────────────────┐
│  ┌──────────────────────────────────────┐ ┌────────┐ │
│  │ 메시지를 입력하세요...               │ │  전송  │ │
│  │                                      │ │        │ │
│  └──────────────────────────────────────┘ └────────┘ │
│                                            0 / 200    │
└──────────────────────────────────────────────────────┘
```

- `textarea` 사용 (멀티라인 지원, `\n` 허용)
- 최대 4줄까지 자동 높이 확장, 4줄 초과 시 내부 스크롤
- `Enter` 전송 / `Shift+Enter` 줄바꿈
- 연결 끊김(재연결 중) 상태에서 `disabled` — `placeholder` 변경: `연결 중...`
- 전송 버튼: `background: #aa3bff; color: white` — 빈 문자열이거나 연결 끊김 시 `disabled`
- 문자 카운터: `{현재} / 200`, 180자 초과 시 주황색(`#f59e0b`), 200자 초과 입력 차단 (DOM 레벨)
- 서버 에러 수신 (`/user/queue/errors`) 시 입력창 하단에 인라인 에러 메시지 (3초 후 자동 소멸)

### 5-7. 연결 상태 배지 (ConnectionStatus) 와이어프레임

```
상태: 연결 중   →  ⬤ 연결 중     (색: #f59e0b — 노란색, 점멸 애니메이션)
상태: 연결됨    →  ● 연결됨     (색: #22c55e — 초록색)
상태: 재연결 중 →  ⬤ 재연결 중  (색: #ef4444 — 빨간색, 점멸 애니메이션)
```

- 점 크기: 8px 원형
- 텍스트: `font-size: 12px; font-weight: 500`
- 점멸: `opacity 1→0.3` 1초 주기 CSS animation (연결 중 / 재연결 중 상태만)

### 5-8. Degraded 배너 (히스토리 조회 실패)

┌────────────────────────────────────────────────────────────┐

## 7. 에러/예외 플로우

### 7-1. 토스트 메시지 정책

- 위치: 화면 상단 중앙 (또는 하단 우측) — 기존 프로젝트 토스트 패턴 있으면 재사용, 없으면 간단히 구현
- 지속 시간: 4초 자동 소멸 (에러는 6초)
- 쌓임: 최대 3개 동시 표시

| 에러 코드 | 토스트 메시지 | 지속 |
|---|---|---|
| EC-8 `ROOM_NOT_FOUND` | 채팅방이 종료되었습니다. | 6초 + `/dbgchat` 이동 |
| EC-11 연결 끊김 | 서버와 연결이 끊어졌습니다. 재연결 중... | 연결 복구 시 자동 소멸 |
| EC-12 등급 강등 | 이 방에 메시지를 보낼 수 없습니다. | 6초 + `/dbgchat` 이동 |
| EC-17 ROOM_DELETED | 채팅방이 종료되었습니다. | 6초 + `/dbgchat` 이동 |
| EC-18 TTL 만료 | 채팅방이 종료되었습니다. | 6초 + `/dbgchat` 이동 |

### 7-2. 방 생성 에러 인라인 표시

- 에러는 토스트가 아닌 `CreateRoomForm` 하단 인라인 표시
- 성공 이후 폼 닫힘 + 즉시 방 진입

### 7-3. 방 목록 로딩 실패

- 로딩 중: 스켈레톤 3개 카드 표시
- 실패 시: `목록을 불러오지 못했습니다. [다시 시도]` — 버튼 클릭 시 재조회

## 8. 연결 상태 전환 다이어그램

```
[DISCONNECTED (초기)]
        |
        | 페이지 마운트
        v
[CONNECTING]  ← 자동 재연결 시도 중
        |
   +----+----+
   |         |
 성공       실패(3회 초과)
   |         |
   v         v
[CONNECTED]  [ERROR — 토스트 + /dbgchat 이동]
   |
   | 연결 끊김 감지
   v
[RECONNECTING]
   |
   | 재연결 3회 시도 (2s / 4s / 8s)
   |
   +-- 성공 --> [CONNECTED]
   |
   +-- 실패 --> [ERROR]
```

- 연결 상태는 `ConnectionStatus` 컴포넌트에 실시간 반영
- `CONNECTED` 상태에서만 메시지 전송 가능 (입력창 활성화)
- `RECONNECTING` 상태: 입력창 비활성화, 배지 빨간 점멸
