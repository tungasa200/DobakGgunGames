## 1. 진입 경로 플로우

```
홈화면 (/)
  └─ 게임 카드 목록 또는 NormalHeader "게임 목록" 드롭다운
       └─ "블록 크러시" 카드 / 메뉴 항목 클릭
            └─ navigate('/block-crush')
                 └─ BlockCrushPage 렌더링
                      └─ idle 상태 (게임 시작 전)
                           └─ "게임 시작" 버튼 클릭
                                └─ playing 상태 전환
```

### 1.1 페이지 레이아웃 상태 흐름

```
/block-crush 접속
  │
  ├─ [idle]
  │    NormalHeader 표시
  │    보드 + 트레이 렌더링 (블록 배치 대기 중)
  │    점수 = 0, 트레이 초기 3블록 채워진 상태
  │    "시작하기" 버튼 표시
  │
  ├─ [playing]
  │    NormalHeader 표시 (자동 숨김 동작 유지)
  │    보드 + 트레이 활성화
  │    HUD: 현재 점수, 콤보 카운터
  │    드래그/드롭 가능 상태
  │
  ├─ [gameOver]
  │    게임 오버 모달 오버레이
  │    배경(보드)은 흐리게 표시 (blur 효과)
  │    최종 점수 / 클리어한 줄 수 표시
  │    닉네임 입력 폼 (기본값: 로그인 시 닉네임)
  │    "등록" 버튼 / "다시 시작" 버튼
  │
  ├─ [rankingInput]
  │    gameOver 모달 내 닉네임 입력 중 상태
  │    입력 검증 (1~12자, 공백 불허)
  │    "등록" 버튼 클릭 → API 호출
  │
  └─ [rankingResult]
       랭킹 등록 완료 모달 (또는 게임 오버 모달 내 전환)
       "내 순위: N위" 표시
       "다시 시작" 버튼 → idle 전환
       "랭킹 보기" 버튼 → 페이지 하단 랭킹 테이블로 스크롤
```

## 3. 상태별 화면 요소 목록

### 3.1 idle 상태

| 영역 | 표시 요소 |
|---|---|
| 헤더 | NormalHeader (accentColor: `--bcr-accent`) |
| 보드 | 8×8 그리드, 모든 셀 빈 상태 |
| 트레이 | 3개 슬롯, 랜덤 폴리오미노 블록 표시 (불투명) |
| HUD | 점수 "0", 콤보 없음 |
| CTA | "게임 시작" 버튼 (보드 위 오버레이 또는 하단) |
| 랭킹 | 페이지 하단 랭킹 테이블 표시 |

### 3.2 playing 상태

| 영역 | 표시 요소 |
|---|---|
| 헤더 | NormalHeader (스크롤 자동 숨김 동작 유지) |
| 보드 | 8×8 그리드, 배치된 블록 / 드래그 미리보기 오버레이 |
| 트레이 | 3개 슬롯 (배치된 슬롯은 반투명 또는 숨김) |
| HUD | 현재 점수 (애니메이션), 콤보 표시 (`COMBO x3`) |
| 줄 클리어 | 클리어 시 해당 행/열 애니메이션 재생 후 제거 |
| 팝업 점수 | 클리어 시 획득 점수 팝업 (보드 위 플로팅) |

### 3.3 gameOver 상태

| 영역 | 표시 요소 |
|---|---|
| 배경 | 보드 + 트레이 (blur 처리, pointer-events: none) |
| 모달 | 반투명 다크 오버레이 위 카드 형태 |
| 모달 상단 | "GAME OVER" 제목 |
| 점수 정보 | 최종 점수 (크게), 클리어한 줄 수 |
| 입력 폼 | 닉네임 입력 필드 + "등록하기" 버튼 |
| 보조 액션 | "랭킹 등록 없이 다시 시작" 텍스트 버튼 |

### 3.4 rankingInput 상태 (gameOver 모달 내 폼 활성)

| 영역 | 표시 요소 |
|---|---|
| 닉네임 필드 | focus 상태, 유효성 메시지 |
| 등록 버튼 | loading 상태 (스피너) |
| 에러 | 유효성 오류 인라인 표시 |

### 3.5 rankingResult 상태

| 영역 | 표시 요소 |
|---|---|
| 결과 메시지 | "등록 완료! 내 순위: N위" 또는 "랭킹 TOP 10 이내 진입!" 강조 |
| 점수 재확인 | 최종 점수 |
| 버튼 | "다시 시작" (주), "랭킹 확인" (보조 — 하단 스크롤) |

## 4. 화면 전환 트리거 상세

| 현재 상태 | 트리거 | 전환 대상 | 전환 효과 |
|---|---|---|---|
| `idle` | "게임 시작" 버튼 클릭 | `playing` | 즉시 전환 (버튼 사라짐) |
| `playing` | 트레이 3블록 모두 배치 불가 판정 | `gameOver` | 300ms fade-in 오버레이 |
| `gameOver` | 닉네임 입력 후 "등록하기" 클릭 | `rankingInput` (로딩) → `rankingResult` | 버튼 → 스피너 → 결과 |
| `gameOver` | "다시 시작" 텍스트 버튼 클릭 | `idle` | 모달 fade-out → 보드 초기화 |
| `rankingResult` | "다시 시작" 클릭 | `idle` | 모달 닫힘 → 보드 초기화 |
| `rankingResult` | "랭킹 확인" 클릭 | 모달 닫힘 + 페이지 하단 스크롤 | smooth scroll |
| 모든 playing | 페이지 새로고침 / 뒤로 가기 | `idle` (게임 상태 초기화) | 브라우저 기본 |

### 4.1 줄/열 클리어 내부 플로우

```
블록 보드에 배치 완료
     │
     ▼
완성된 행 / 열 감지 (동시에 여러 개 가능)
     │
     ├─ 없음: 트레이 슬롯 null 처리 → 새 블록 필요하면 트레이 보충
     │
     └─ 있음:
          줄 클리어 애니메이션 재생 (페이드아웃 / 플래시)
          점수 계산 (줄 수 × 기본 점수 × 콤보 배수)
          콤보 카운터 증가
          팝업 점수 표시 (150ms 후 fade-out)
          클리어된 셀 제거
          트레이 슬롯 해당 항목 null 처리
          트레이 3개 모두 null이면 새 세트 생성
     │
     ▼
배치 가능 여부 재판정
  ├─ 가능: playing 유지
  └─ 불가: gameOver 전환
```

## 5. 인터랙션 — PC(마우스) vs 모바일(터치)

### 5.1 블록 드래그 공통 플로우

```
[트레이 슬롯에서 블록 집어들기]
  → 드래그 시작: 원본 슬롯 반투명 유지 (opacity: 0.35)
  → 드래그 아이템: 블록 폴리오미노 형태 커서/손가락 따라 이동
  → 보드 위 진입: 셀 스냅(grid-align) 미리보기
       → 유효 위치: 해당 셀들 초록/강조 반투명 색상
       → 무효 위치(이미 채워짐): 해당 셀들 붉은 반투명 색상
  → 드롭(마우스 업 / 터치 엔드):
       → 유효 위치: 블록 배치 확정 → 원본 슬롯 null
       → 무효 위치 또는 보드 외부: 원본 슬롯 복원 (opacity: 1)
```

### 5.2 PC(마우스) 전용 동작

| 동작 | 설명 |
|---|---|
| `mousedown` on 트레이 슬롯 | 드래그 시작, `cursor: grabbing` |
| `mousemove` | 드래그 아이템 위치 추적, 보드 셀 스냅 계산 |
| `mouseup` (보드 위) | 배치 확정 또는 취소 |
| `mouseup` (보드 외부) | 드래그 취소, 원본 슬롯 복원 |
| hover (보드 위 드래그 없음) | 미리보기 없음 (드래그 중에만 표시) |

### 5.3 모바일(터치) 전용 동작

| 동작 | 설명 |
|---|---|
| `touchstart` on 트레이 슬롯 | 드래그 시작, 햅틱 피드백 (선택) |
| `touchmove` | 드래그 아이템 위치 추적, `touch-action: none` 필수 |
| `touchend` (보드 위) | 배치 확정 또는 취소 |
| `touchcancel` | 드래그 취소, 원본 슬롯 복원 |
| 터치 영역 | 트레이 슬롯 최소 터치 영역: 44×44px 이상 |

### 5.4 모바일 전용 UX 고려사항

- 드래그 중 화면 스크롤 방지: `touch-action: none` on 드래그 대상
- 드래그 아이템은 손가락 위(y offset -80px 권장)에 표시하여 배치 위치가 손가락에 가려지지 않도록
- 보드 셀 크기가 작을 경우(모바일) 터치 정확도 보완을 위해 인접 셀 스냅 허용 (±0.5셀 오차 허용)

## 6. 랭킹 등록 플로우

```
[gameOver 상태 모달]
      │
      ├─ 로그인 유저:
      │    닉네임 기본값 = 계정 닉네임 (pre-fill)
      │    수정 가능
      │
      └─ 비로그인(게스트):
           닉네임 필드 비어있음, 직접 입력 필요

[닉네임 입력 검증]
  - 1~12자 이내
  - 공백만으로 구성 불가
  - 금지어 필터 적용 (shared/badwords.json 준수)
  - 실패: 인라인 에러 메시지, 버튼 비활성

[등록하기 클릭]
      │
      ▼
  POST /api/rankings
  { gameKey: 'block-crush', name: '닉네임', score: 점수 }
      │
      ├─ 성공 (200):
      │    rankingResult 상태 전환
      │    내 순위(rank) 표시
      │
      └─ 실패 (4xx/5xx):
           에러 토스트 표시 ("등록에 실패했습니다. 다시 시도해 주세요.")
           모달 유지, 재시도 가능
```

## 7. 엣지케이스 UX 처리

| ID | 상황 | UX 처리 |
|---|---|---|
| EC-1 | 트레이 3개 중 일부만 배치 가능한 상태 | 배치 불가 슬롯만 시각적으로 비활성(opacity 낮게), 배치 가능 슬롯은 정상 표시 |
| EC-2 | 드래그 중 브라우저 탭 전환 / 포커스 잃음 | `blur` 이벤트 감지 → 드래그 취소, 원본 슬롯 복원 |
| EC-3 | 동시에 여러 줄/열 클리어 | 모두 동시에 클리어 애니메이션, 점수는 합산 후 1회 팝업 표시 |
| EC-4 | 콤보 중 게임 오버 조건 충족 | 클리어 애니메이션 완료 후 게임 오버 판정 (애니메이션 도중 판정 안 함) |
| EC-5 | 랭킹 API 응답 지연 (>3초) | 스피너 유지, 타임아웃 없음 (사용자 대기) |
| EC-6 | 네트워크 오류로 랭킹 등록 실패 | 에러 토스트 + 재시도 버튼, 게임 데이터는 유지 (다시 시작 전까지) |
| EC-7 | 보드 전체가 꽉 찬 상태에서 트레이 보충 | 트레이 보충 시 자동으로 게임 오버 판정 |
| EC-8 | 페이지 새로고침 | 게임 상태 초기화 (저장하지 않음), idle 상태로 시작 |
