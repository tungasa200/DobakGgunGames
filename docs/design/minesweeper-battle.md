## 1. 색상 팔레트

### 1.1 디자인 원칙
- 솔로 지뢰찾기(`MinesweeperBoard.module.css`)의 라이트 톤을 기반으로 배틀 요소만 다크 네온 오버레이로 강조한다.
- Blockfall Battle(`blockfall-battle.css`)의 인디고 네온 어휘를 지뢰찾기 배틀에도 동일하게 적용하여 브랜드 일관성을 유지한다.
- 솔로 모드는 밝은 배경에서 플레이하지만, 배틀 모드는 경쟁감을 위해 다크 배경을 사용한다.

## 3. 컴포넌트 스타일 명세

### 3.1 보드 셀 크기

| 환경 | 셀 크기 | 근거 |
|---|---|---|
| 데스크톱 playing | `28px` | 솔로 30px에서 2px 축소 (사이드패널 공간 확보) |
| 데스크톱 ready | `30px` | 솔로 동일 (보드 전체 너비 사용) |
| 모바일 (600px 이하) | `26px` | 9셀 × 26px = 234px, 모바일 화면 내 수용 |
| 폰트 크기 (숫자) | `13px` (28px 셀) / `15px` (30px 셀) | 솔로 기준에서 비례 |

### 3.2 사이드패널 — 상대 프로그레스 바

```
상대 진행률 바 (OpponentProgress 컴포넌트)
+----------------------------------+
| {닉네임}              [상대]     |
+----------------------------------+
| [████████████░░░░░░░]            |
|  32%          (23 / 71 셀)       |
+----------------------------------+
```

- 바 컨테이너 높이: `8px`, 배경: `--msb-border-default`
- 바 채움: `--msb-opponent-accent` (#F59E0B, 앰버)
- transition: `width 200ms ease-out` (throttle 주기와 맞춤)
- 본인 진행률 바: `--msb-accent` (인디고) 색으로 구별

## 7. 접근성

### 7.1 키보드 네비게이션
- idle/waiting/ready/finished 화면의 모든 버튼: Tab 순서 확보
- 지정 셀 (4,4): `aria-label="시작 셀 — 클릭하여 게임 시작"` 추가
- 포기 확인 모달: `role="dialog"`, `aria-modal="true"`, ESC 키로 닫기 가능

### 7.2 색상 대비
- `--msb-text-primary` (#f0f6fc) / `--msb-bg-panel` (#1c2128): 대비비 약 14:1 (WCAG AAA)
- `--msb-highlight-cell` (#EAB308) on `#78350F`: 충분한 대비 확보
- 상태 인디케이터: 색상만으로 구별하지 않고 텍스트(✓/⏳) 병기

### 7.3 aria 속성
- 프로그레스 바: `role="progressbar"`, `aria-valuenow`, `aria-valuemin="0"`, `aria-valuemax="100"`
- 타이머: `aria-live="polite"` (초당 갱신이 아닌 5초 주기로 aria 갱신 권장)
- 연결 끊김 배너: `role="alert"`
