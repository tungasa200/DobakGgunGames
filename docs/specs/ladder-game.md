## 1. 배경 및 목적

### 배경
- DobakGgun Games 는 솔로플레이 미니게임 컬렉션으로, 현재 랭킹 게임 6종(지뢰찾기/숫자야구/블록폴/솔리테어/사과게임/스도쿠) + 미니게임 도구 2종(룰렛 `/roulette`, 주사위 `/dice`) 을 운영 중.
- 미니게임 도구는 **서버/DB/랭킹 없이 순수 클라이언트사이드**로 동작하며, 친구/가족과 모였을 때 즉석에서 사용하는 파티 도구로 자리잡고 있음.
- 한국 전통 파티 게임인 **사다리 타기**를 추가하여 미니게임 도구 라인업을 강화하고, 기존 룰렛/주사위와 함께 "오프라인 모임에서 의사결정 도구"로 활용성을 확장.

### 목적
- 누구나 로그인 없이 `/ladder` 라우트로 진입해 사다리 타기를 즉시 플레이할 수 있는 **순수 클라이언트 도구** 추가.
- 참가자 이름과 결과(상품/벌칙)를 입력 → 랜덤 사다리 생성 → 경로 시각화 → 매칭 결과 확인의 전체 플로우 제공.
- 기존 `/roulette`, `/dice` 와 동일한 UX 결을 따라 HomePage "미니게임" 카드 영역에 신규 링크 추가.

### 비목표 (Out of Scope)
- 백엔드 연동, DB 저장, 랭킹 시스템 통합.
- 회원 기능(로그인/저장/공유 링크 등) — 결과 영속화는 본 PRD 범위 외.
- Excel 모드 — 사용자 지시 없음 (§3 참조).
- 멀티플레이/실시간 동기화 (혼자 또는 한 디바이스를 돌려보는 형태로 충분).
- 온라인 공유 / 결과 이미지 다운로드 (Nice-to-have 후순위).

## 2. 유저 스토리

- **US-1 (핵심)** — As a player, I want to enter participants' names and outcomes, so that I can run a quick lottery for who gets what.
- **US-2** — As a player, I want the ladder to be generated randomly each time, so that the result feels fair and unpredictable.
- **US-3** — As a player, I want to see each participant's path traced visually down the ladder, so that everyone can verify the result is honest.
- **US-4** — As a player, I want to choose between revealing all results at once or one by one, so that I can build suspense at a party.
- **US-5** — As a player, I want a reset button, so that I can re-run the same game with new randomness without re-entering everything.
- **US-6** — As a player, I want to log in or not — either should work — so that I can use the tool without friction.
- **US-7** — As a player accessing from the home page, I want to find the ladder game next to roulette and dice, so that I can see all party tools in one place.

## 5. 게임 규칙 / 점수 정책

### 5.1 사다리 구조
- **세로 선 (rails)**: 참가자 수 N 만큼 (예: 4명 → 세로 선 4개).
- **가로 선 (rungs)**: 인접한 두 세로 선 사이를 잇는 짧은 선. 무작위 배치.
- **행(row) 개수**: 디자이너/개발자가 시각적 균형을 보고 결정 (제안: N에 비례, 예: 행 = N × 4 정도).
- **무결성 규칙**: 같은 행에서 한 세로 선이 좌·우 양쪽 모두 가로 선과 연결되면 안 됨 (이중 분기 금지). 알고리즘이 이 제약을 보장해야 함.

### 5.2 매칭 규칙
- 각 참가자는 자신의 세로 선 상단에서 출발 → 아래로 진행 → 가로 선을 만나면 좌/우로 이동 → 다시 아래로 → 바닥의 결과 셀에 도달.
- 모든 매칭은 **유일성 보장** (서로 다른 참가자가 서로 다른 결과로 매핑) — 사다리 무결성으로 자동 충족.
- 가로 선 배치는 의사난수(`Math.random()` 사용 가능, 보안 요구 없음).

### 5.3 점수/타이머/레벨
- **점수 없음** — 단순 매칭 도구.
- **타이머 없음** — 사용자 페이스로 진행.
- **레벨 없음**.
- **승패 없음** — 결과는 운에 따른 매칭 (재미를 위한 도구).

### 5.4 진행 흐름
1. 페이지 진입 → 기본 참가자 4명/결과 4개로 초기화 (예시 텍스트 자리표시자).
2. 사용자가 참가자/결과 텍스트와 인원 수 조정.
3. 결과 공개 모드(A/B) 선택.
4. "사다리 타기" 버튼 클릭 → 사다리 시각화 + 매칭 계산.
5. 모드 A: 전체 매칭 결과 즉시 표시 / 모드 B: 참가자 클릭 시 경로 애니메이션 → 결과 1건 공개.
6. "초기화" 버튼으로 새 사다리 다시 생성 (참가자/결과 입력 유지).

## 12. 게임 밸런스 / 난이도 정책

- 본 게임은 **운(luck) 기반 도구**로 점수/난이도 개념 없음.
- 가로 선 밀도(FR-S1)만 "재미" 변수:
  - 너무 적으면 결과가 직관적으로 예측 가능 → 재미 감소.
  - 너무 많으면 사다리가 어지러워 시각적 피로.
  - **잠정 정책**: "행 수 = 참가자 수 × 4, 각 행에서 가로 선 1~2개를 무결성 규칙 하에 무작위 배치" — 디자이너/개발자가 POC 후 미세 조정.
