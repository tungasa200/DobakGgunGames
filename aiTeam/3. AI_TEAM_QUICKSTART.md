## 시작 전에

### 이 가이드를 끝내면 뭐가 생기나요?

집 PC든 회사 PC든 어디서나 Claude Code를 켜고 `/resume` 한 줄이면,
기획자·디자이너·프론트엔드 개발자·백엔드 개발자·QA 테스터 5명의 AI 팀이
어제 하던 작업을 이어받아 일을 시작합니다.

### 준비물

- [ ] DobakGgun 프로젝트가 열려있는 **PC**
- [ ] **VS Code** 설치됨
- [ ] **Node.js 18 이상** 설치됨 (모르면 다음 단계에서 확인)
- [ ] **Claude Max 플랜** 계정
- [ ] **Git** 설정됨 (지금까지 커밋하고 푸시하고 있었으면 OK)
- [ ] `AI_TEAM_SETUP_final.md` 파일 (이전에 만든 최종 문서)

### 걸리는 시간

- 집 PC 최초 세팅: **약 30분**
- 회사 PC에 복제하기: **약 10분**
- 첫 테스트 (리뷰 팀 실행): **약 15분**

### 용어 미리보기 (몰라도 진행 가능)

| 용어 | 쉽게 말하면 |
|---|---|
| Claude Code | 터미널에서 쓰는 Claude. 파일을 직접 읽고 쓸 수 있음 |

| 팀원 (teammate) | 팀 안에 있는 각각의 Claude. 역할이 정해져 있음 |
| 리드 (lead) | 팀장 역할의 Claude. 당신이 이 친구에게 지시 |
| in-process 모드 | 팀원 5명이 하나의 터미널 창 안에 있는 방식 |
| `Shift+Down` | 팀원끼리 전환하는 단축키 |

## 1단계 — Claude Code 준비하기

### 목표

내 PC에 Claude Code가 설치되어 있고, Max 플랜 계정으로 로그인되어 있는지 확인합니다.

### 1-1. VS Code 열기

원래 하던 대로 VS Code를 켭니다.

### 1-2. DobakGgun 프로젝트 열기

`파일 → 폴더 열기`로 프로젝트 폴더(예: `dobakggun`)를 엽니다.
이미 열려있으면 그대로 진행.

### 1-3. 터미널 열기

VS Code 상단 메뉴에서 `터미널 → 새 터미널` (단축키: `Ctrl + `` `)
화면 아래쪽에 까만 창이 하나 열립니다. 앞으로 여기에 명령어를 입력할 거예요.

### 1-4. Node.js 확인

터미널에 이 명령어를 **복사 → 붙여넣기 → Enter**:

```bash
node --version
```

**화면에 이렇게 나오면 성공:**
```
v20.11.0
```
(숫자는 다를 수 있어요. **v18 이상**이면 OK)

**`node를 찾을 수 없다`는 에러가 나면:**
→ Node.js가 설치 안 되어 있습니다. https://nodejs.org 에서 LTS 버전 다운로드 후 설치.

### 1-5. Claude Code 설치 (또는 업데이트)

터미널에:

```bash
npm install -g @anthropic-ai/claude-code
```

설치에 **1~3분 정도** 걸립니다. 기다려주세요.
마지막에 별다른 에러 없이 프롬프트(`>` 또는 `PS>`)가 돌아오면 성공.

### 1-6. Claude Code 버전 확인

```bash
claude --version
```

**화면에 이렇게 나오면 성공:**
```
2.1.32  (또는 더 높은 숫자)
```

**반드시 2.1.32 이상**이어야 합니다. 낮으면 위의 1-5를 다시 실행하세요.

### 1-7. 로그인

```bash
claude login
```

브라우저가 자동으로 열리며 Claude 로그인 화면이 나옵니다.
**Max 플랜으로 가입된 이메일**로 로그인 → 승인 → 터미널로 돌아오면 끝.

**이 단계의 성공 체크리스트:**
- [ ] `claude --version`이 2.1.32 이상
- [ ] `claude login`으로 로그인 완료

## 3단계 — Claude Code로 팀 설정 파일 만들기

### 목표

Claude Code에게 `AI_TEAM_SETUP_final.md`를 읽게 하고, 거기 쓰여있는 대로
`CLAUDE.md`, `.claude/agents/*.md` 등 실제 파일들을 만들게 합니다.

### 3-1. Claude Code 실행

터미널에:

```bash
claude
```

화면에 Claude 로고가 나오고 프롬프트 입력창이 나타납니다.
이제부터 자연어로 대화하면 됩니다.

### 3-2. 설정 파일 생성 요청하기

아래 텍스트를 **통째로 복사** 해서 붙여넣고 Enter:

```
AI_TEAM_SETUP_final.md 파일을 읽어줘.

그다음 섹션 A의 A-3 ~ A-8을 순서대로 실행해서 실제 파일들을 만들어줘:

1. CLAUDE.md 생성 (프로젝트 루트)
2. .claude/agents/ 아래 5개 파일 생성:
   - planner.md
   - designer.md
   - developer-frontend.md
   - developer-backend.md
   - qa-tester.md
3. .claude/commands/resume.md 생성
4. .claude/settings.json 생성
5. docs/ 아래 4개 하위 폴더(specs, design, progress, review) 생성 + 각 .gitkeep
6. .gitignore에 Claude 관련 항목 추가

주의사항:
- 기존에 CLAUDE.md나 .claude/ 디렉토리가 있으면 덮어쓰기 전에 나한테 보여주고 확인받을 것
- 기존 .gitignore가 있으면 끝에 append만 하고, 기존 내용 덮어쓰지 말 것
- 각 파일 내용은 AI_TEAM_SETUP_final.md에 쓰여있는 그대로 충실히 반영

완료하면 생성된 파일 목록을 알려줘.
```

### 3-3. Claude가 작업하는 동안 지켜보기

Claude가 파일을 하나씩 만드는 과정이 보입니다. **2~5분** 정도 걸릴 수 있어요.

중간에 이런 질문이 나올 수 있습니다:

- **"CLAUDE.md가 이미 있는데 덮어쓸까요?"** → 기존 파일 내용을 보고 판단. 지금 생성될 내용과 병합이 필요하면 "내용 보여줘" 라고 요청 후 수동 조정
- **"파일 생성 권한을 주시겠습니까?"** → `Yes, always` 또는 `승인` 선택

### 3-4. 작업 완료 확인

Claude가 "완료했습니다" 비슷한 메시지와 함께 생성 파일 목록을 보여주면 성공입니다.

**이 단계의 성공 체크리스트:**
- [ ] 에러 없이 작업이 끝남
- [ ] Claude가 생성한 파일 목록이 화면에 나옴

## 4단계 — 파일이 잘 만들어졌는지 확인

### 목표

눈으로 직접 확인해서 뭐가 잘못 안 됐는지 체크합니다.

### 4-1. VS Code 왼쪽 탐색기에서 확인

다음 폴더/파일들이 생겼는지 하나씩 체크:

```
dobakggun/
├── CLAUDE.md                              ← 있어야 함
├── .claude/                               ← 있어야 함
│   ├── agents/
│   │   ├── planner.md                     ← 있어야 함
│   │   ├── designer.md                    ← 있어야 함
│   │   ├── developer-frontend.md          ← 있어야 함
│   │   ├── developer-backend.md           ← 있어야 함
│   │   └── qa-tester.md                   ← 있어야 함
│   ├── commands/
│   │   └── resume.md                      ← 있어야 함
│   └── settings.json                      ← 있어야 함
└── docs/
    ├── specs/      (.gitkeep 들어있음)
    ├── design/     (.gitkeep 들어있음)
    ├── progress/   (.gitkeep 들어있음)
    └── review/     (.gitkeep 들어있음)
```

하나라도 빠져있으면 Claude에게 말하세요:

```
[빠진 파일명]이 안 만들어진 것 같아. 확인하고 만들어줘.
```

### 4-2. 파일 내용 대충 훑어보기

그냥 믿고 넘어가도 되지만, 한 번은 확인하는 게 좋아요.

1. **CLAUDE.md** 열어서: "Project: DobakGgun Games" 같은 프로젝트 이름이 맞게 써있는지
2. **`.claude/agents/planner.md`** 열어서: 내 프로젝트(게임 관련) 내용이 들어있는지
3. **`.claude/settings.json`** 열어서: 아래처럼 생겼는지 확인

```json
{
  "env": {
    "CLAUDE_CODE_EXPERIMENTAL_AGENT_TEAMS": "1"
  }
}
```

이게 가장 중요해요. 이 설정이 있어야 팀 기능이 활성화됩니다.

### 4-3. `.gitignore` 확인

`.gitignore` 파일을 열어서 **맨 아래에** 이 내용이 추가되어 있는지 확인:

```gitignore
# Claude Code local state
.claude/settings.local.json
.claude/logs/
.claude/cache/
```

없으면 직접 추가하세요.

**이 단계의 성공 체크리스트:**
- [ ] 모든 파일/폴더가 위치에 존재
- [ ] `.claude/settings.json`에 `CLAUDE_CODE_EXPERIMENTAL_AGENT_TEAMS` 있음
- [ ] `.gitignore`에 Claude 항목 추가됨

```bash
ls .claude/agents
```

## 7단계 — 첫 테스트 (안전한 리뷰 팀)

### 목표

진짜 기능 개발을 하기 전에 팀이 잘 돌아가는지 **망가질 수 없는 작업**으로 테스트합니다.

### 7-1. Claude Code 실행

터미널에:

```bash
claude
```

### 7-2. 리뷰 팀 생성 프롬프트

아래 텍스트를 통째로 복붙하고 Enter:

```
Create an agent team to review our main branch.
Spawn 3 teammates (임시 역할, .claude/agents/ 파일과 무관):
- One on architecture smells (순환 의존, 비대한 컴포넌트, 중복 게임 로직)
- One on security concerns (JWT 처리, 환경변수 노출, HMAC 검증)
- One on test coverage gaps (프론트 + 백엔드 테스트 누락 영역)

각자 docs/review/{category}.md에 findings 저장하고,
완료되면 서로 공유해서 우선순위 통합 목록 하나 만들어줘 (docs/review/priorities.md).
```

### 7-3. 권한 요청 승인

```
Do you want to allow creating an agent team? (y/N)
```

이 질문이 나오면 `y` 입력 후 Enter.

### 7-4. 팀원들이 일하는 걸 구경하기

화면에 3명의 팀원이 각각 코드를 읽고 분석하는 과정이 나타납니다.
**5~10분 정도** 걸려요.

### 7-5. 단축키 연습해보기

작업 진행 중에 다음을 시도해보세요:

- **`Shift + Down`** — 팀원 간 전환. 눌러보면 "지금 어느 팀원을 보고 있는지" 바뀜
- **`Ctrl + T`** — 태스크 리스트 토글. 누가 뭐 하고 있는지 한눈에 봄
- **`Esc`** — 현재 팀원의 턴 중단

### 7-6. 팀원에게 직접 말 걸어보기

`Shift + Down`으로 security 리뷰어 선택 → 직접 타이핑:

```
JWT 토큰 만료 시간도 확인해줘
```

특정 팀원한테만 추가 지시가 들어갑니다.

### 7-7. 작업 완료 확인

모든 팀원이 "idle" 상태가 되고, `docs/review/` 아래에 파일들이 생겼는지 확인:

```
docs/review/
├── architecture.md
├── security.md
├── test-coverage.md
└── priorities.md
```

### 7-8. 팀 정리

Claude Code 입력창에:

```
Clean up the team
```

리드가 "팀을 정리했습니다"라고 답하면 성공입니다.

**이 단계의 성공 체크리스트:**
- [ ] 3명의 팀원이 스폰되어 각자 리뷰함
- [ ] `Shift+Down`으로 팀원 전환 가능
- [ ] `docs/review/` 아래 파일 4개 생성됨
- [ ] `Clean up the team`이 정상 작동함

## 실전 — 진짜 기능 개발 시작하기 (팀이 익숙해진 뒤)

### 기본 템플릿 (복사해서 수정)

리뷰 팀 테스트가 잘 됐으면, 이제 실전입니다.


## 자주 겪는 문제

### 문제 1: `CLAUDE_CODE_EXPERIMENTAL_AGENT_TEAMS` 를 인식 못 한다는 메시지

**원인**: Claude Code 버전이 낮음

**해결**:
```bash
npm install -g @anthropic-ai/claude-code
claude --version
```
2.1.32 이상인지 확인.

### 문제 2: "팀을 만들 수 없다"고 나옴

**원인**: `.claude/settings.json`에 환경변수 설정이 안 됨

**해결**: `.claude/settings.json`을 VS Code에서 열어서 내용이 아래와 같은지 확인:

```json
{
  "env": {
    "CLAUDE_CODE_EXPERIMENTAL_AGENT_TEAMS": "1"
  }
}
```

아니면 직접 편집.

### 문제 3: `Shift + Down`을 눌러도 아무 반응이 없음

**원인 A**: 팀이 아직 안 만들어짐

→ 먼저 `Create an agent team...` 프롬프트로 팀을 만들어야 작동합니다.

**원인 B**: 다른 프로그램이 단축키를 가로챔

→ VS Code 단축키 설정에서 `Shift+Down`이 뭐에 할당돼있는지 확인.

### 문제 4: 팀원 중 한 명이 에러로 멈춤

**해결**: 리드에게 말하기:

```
[멈춘 팀원 이름] 대신 새 팀원을 스폰해서 이어가게 해
```

### 문제 5: 권한 요청이 너무 자주 뜸

**해결**: 자주 쓰는 동작은 한 번에 영구 허용. 첫 요청 때 `Yes, always allow` 같은 옵션을 선택.

### 문제 6: 한 PC에서 팀 정리 안 하고 나갔더니 다른 PC에서 이상함

**원인**: 실험 기능 특성상 세션 상태가 꼬임

**해결**:
1. 이전 PC에서 Claude Code 다시 열고 `Clean up the team`
2. 그게 안 되면 그냥 무시하고, 새 PC에서 `/resume`로 새 팀 스폰 (예전 팀 상태는 PC에 남아있지만 실제로는 유령이라 문제 없음)

### 문제 7: `.env` 관련 변경이 필요하다고 팀원이 말함

**상황**: 팀원이 "환경변수 XXX를 추가해야 한다"고 보고함

**해결 절차**:
1. 프론트엔드 변수 → **Vercel 대시보드 → 프로젝트 → Settings → Environment Variables**에 추가
2. 백엔드 변수 → **Railway 대시보드 → 프로젝트 → Variables**에 추가
3. 추가 후 Vercel/Railway에서 재배포 트리거
4. Claude에게 "환경변수 추가하고 재배포했어, 다음 단계 진행해"

### 문제 8: 빌드/배포가 실패했을 때

**해결 절차**:
1. Vercel 또는 Railway 대시보드에서 **실패한 빌드 로그 전체 복사**
2. Claude Code에 붙여넣으며:

```
빌드가 실패했어. 로그는 다음과 같아:

[로그 붙여넣기]

담당 developer에게 전달해서 원인 분석하고 수정해줘.
```

3. 수정 후 `git push` → 재배포 결과를 다시 알림

### 문제 9: 팀 작업이 너무 느림 / 토큰 낭비 같음

**해결 체크리스트**:
- 팀원 수가 너무 많은가? (3~5명이 적절)
- 작업이 너무 작은데 5명 다 띄웠는가? → 2~3명으로 축소
- 진행 상태를 30분마다 점검하고 있는가?
  → `지금 각 팀원 상태 요약해줘`로 중간 점검

### 문제 10: 2시간 넘게 한 세션이 돌고 있음

**해결**: 세션을 길게 끌지 말기

1. 현재 상태에서 팀원들에게 `docs/progress/` 업데이트 지시
2. `Clean up the team`
3. Git 커밋 & 푸시
4. 잠깐 쉬고 새 세션 시작 → `/resume`으로 이어받기

## 내일을 위한 체크리스트

작업 끝낼 때마다 이 순서로:

- [ ] 팀원들이 모두 idle인지 확인
- [ ] 팀원들에게 `docs/progress/` 업데이트 지시
- [ ] `Clean up the team` 명령
- [ ] Claude Code 종료 (`/exit`)
- [ ] `git add .`
- [ ] `git commit -m "feat/fix: 작업 내용"`
- [ ] `git push`

작업 시작할 때마다 이 순서로:

- [ ] 터미널에서 `git pull`
- [ ] `claude` 실행
- [ ] `/resume` 입력
- [ ] 팀이 어제 한 일을 파악하고 이어갈 때까지 대기
- [ ] 오늘 할 일 지시
